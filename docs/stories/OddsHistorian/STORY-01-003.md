## User Story

**As a** sistema de coleta (scraper)  
**I want to** persistir os dados brutos de odds coletados em tempo real  
**So that** eu tenha um histórico temporal para treinamento de modelos e auditoria.

## Description

Esta story implementa a camada de repositório para salvar os snapshots de odds. Sempre que um scraper termina uma rodada de coleta para uma partida, os valores de 1x2 devem ser salvos na tabela `odds_snapshot`. É fundamental garantir a integridade referencial com a tabela `matches`. Se a partida ainda não existir no banco, ela deve ser criada primeiro (ou no mesmo fluxo).
