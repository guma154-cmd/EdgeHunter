# Architecture Decision Records (ADRs)

Este diretório contém os ADRs (Architecture Decision Records) do projeto EdgeHunter, de acordo com o método BMAD.

## O que é um ADR?
É um documento curto que captura uma decisão arquitetural importante tomada com relação a um sistema de software, junto com seu contexto e as consequências dessa decisão.

## Formato
Siga o template padrão BMAD para ADRs:
- **Title** (O que estamos decidindo)
- **Status** (Proposed / Accepted / Deprecated)
- **Context** (O que está forçando a decisão)
- **Decision** (A decisão final tomada)
- **Consequences** (O que se torna mais fácil ou mais difícil após a decisão)
