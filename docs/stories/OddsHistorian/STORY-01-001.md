## User Story

**As a** desenvolvedor do sistema  
**I want to** ter um mecanismo de inicialização de schema SQL que possa ser executado múltiplas vezes sem erro  
**So that** a implantação e atualização do banco de dados sejam confiáveis e automatizadas em diferentes ambientes.

## Description

Esta story foca na criação da fundação de dados do projeto. O SQLite foi escolhido (ADR-004) por sua simplicidade. Precisamos de um script SQL robusto que defina as tabelas `odds_snapshot`, `matches` e `bookies` (se necessário), garantindo que a execução subsequente do script não apague dados existentes ou falhe porque as tabelas já existem.

O componente Python deve carregar este script e executá-lo durante a inicialização do container ou da aplicação. Isso evita a necessidade de migrações manuais complexas no estágio inicial do protótipo.

### Code Structure
```python
# backend/app/db/database.py
import sqlite3
import os

class DatabaseManager:
    def __init__(self, db_path: str):
        self.db_path = db_path

    def init_db(self, schema_path: str):
        with sqlite3.connect(self.db_path) as conn:
            with open(schema_path, 'r') as f:
                conn.executescript(f.read())
```

### Database Changes (if any)
- Tabela `matches`: match_id (PK), home_team, away_team, league, date, status.
- Tabela `odds_snapshot`: id (PK), match_id (FK), bookie, odds_home, odds_draw, odds_away, timestamp.
- Tabela `simulated_bets`: id (PK), match_id (FK), opportunity_id, market, selection, odds, stake, result, profit_loss, roi, created_at, settled_at. Índices em `match_id` e `created_at`.
